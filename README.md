# To-Do-Webanwendung (HTML, CSS, JavaScript)

## Projektübersicht

Dies ist eine To-Do-Listen-Anwendung, die vollständig mit **Vanilla HTML, CSS und JavaScript** (ohne Frameworks) entwickelt wurde. Sie dient als praktisches Hochschulprojekt, um Grundkenntnisse in der Webentwicklung und im **Versionskontrollsystem Git/GitHub** unter Beweis zu stellen.

Die Anwendung ist clientseitig implementiert und verwendet den **Local Storage** des Browsers zur dauerhaften Speicherung der Aufgaben.

---

## Anforderungen & Implementierte Funktionen

Die Anwendung erfüllt die folgenden Kernfunktionen, die über die reine Aufgabenverwaltung hinausgehen:

### 1. Grundlegende CRUD-Operationen (Create, Read, Update, Delete)

| Funktion | Beschreibung |
| :--- | :--- |
| **Hinzufügen** | Erstellen neuer Aufgaben über ein Eingabefeld und einen Button. |
| **Lesen** | Anzeige aller Aufgaben in einer übersichtlichen Liste. |
| **Löschen** | Entfernen einzelner Aufgaben über einen dedizierten "Löschen"-Button. |
| **Erledigt-Markierung** | Aufgaben können per Klick als **erledigt** markiert werden. Erledigte Aufgaben erhalten eine optische Kennzeichnung (durchgestrichen und ausgegraut). |

### 2. Organisation & Sortierung

| Funktion | Beschreibung |
| :--- | :--- |
| **Prioritätskennzeichnung** | Neue Aufgaben können mit den Prioritäten **"Hoch"**, **"Mittel"** oder **"Niedrig"** versehen werden. |
| **Visuelle Priorität** | Jede Aufgabe erhält eine farbige Markierung am linken Rand basierend auf ihrer Priorität. |

### 3. Persistenz & Filterung

| Funktion | Beschreibung |
| :--- | :--- |
| **Lokale Speicherung** | Alle Aufgaben (inkl. Text, Status und Priorität) werden im **Browser Local Storage** gespeichert und beim Neuladen der Seite wiederhergestellt. |
| **Filterung** | Es stehen drei Filter-Buttons zur Verfügung, um die Aufgabenliste dynamisch einzuschränken: **"Alle"**, **"Offen"** und **"Erledigt"**. |
